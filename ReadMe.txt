这个程序起名为FinalSensor显然是在完成最后一步:   写Uart和识别上位机发送的命令这部分程序.

现在已经知道并且验证:
                                 以十进制发送的时候PC发给单片机的是2个字节,  而字符形式和16进制形式则是1个字节
    
这里关键的一点是区分10进制16进制和字符形式
                                 1---  16进制就是十六进制,  51单片机使用的其实就是16进制  其他的都是转为16进制的
                                 2---   10进制这个专门指代PC上位机的串口调试软件以10进制发送给下位机, 它具有两个                                                                      字节,也就是说, 连续发送两个字节给单片机
                                 3---    字符形式  在某些串口软件中如果不勾选16进制那么就是字符形式,如串口调试器                                                                  软件.而有些软件则是10进制. 这个为关键点.   

猜测:    
            上位机软件以字符形式发送给单片机时其实发的也是16进制的数(正确的说应该是二进制,因为单片机只             认二进制),  所以是一个字节,(ASCII码由一个字节表示, 而字符由ASCII码取得,因此字符占1个字节).
